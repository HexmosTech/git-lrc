# Release v0.3.11

Date: 2026-05-26

## Summary
- Fix LiveReview hook behavior in linked git worktrees so valid attestations are honored during commit, resolving GitHub issue #89.
- Fix local hook-management commands in linked worktrees so install, uninstall, and status resolve the correct repository hook paths.

## Install and Update
New install:
```
# Linux/macOS
curl -L https://hexmos.com/ipm-install | bash && ipm i HexmosTech/git-lrc

# Windows
iwr https://hexmos.com/ipm-install-ps | iex; ipm i HexmosTech/git-lrc
```

Update:

```
lrc self-update
```

## Changes
- Managed hooks now resolve attestation files, disabled markers, and related hook state from the active worktree git dir instead of assuming `.git` is a directory. This fixes `lrc review --staged --skip`, `--vouch`, and normal commit flow inside linked worktrees created with `git worktree add`, addressing GitHub issue #89.
- `lrc hooks install --local`, `lrc hooks uninstall --local`, and `lrc hooks status` now work correctly when run from inside a linked worktree by resolving the worktree root, per-worktree git dir, and shared git common dir explicitly.
- Release and preflight targets now enforce the storage and network status-doc link check before publishing, so release automation cannot bypass the Rule 2 documentation requirement.

1. VS Code core already has a chat session storage concept under workspace storage, including chatSessions plus special cases for empty-window and transferred sessions.
2. On this machine, Copilot chat transcripts are concretely present as per-workspace JSONL files under the VS Code user-data workspace storage for the GitHub Copilot Chat extension.

4. The Copilot transcript payload can include visible content plus tool requests and assistant reasoning metadata, so CommonChat needs a normalized layer and a raw provenance layer.
5. The cleanest first cut is a filesystem-read-only extractor. That avoids premature database or network coupling while the source formats are still being mapped.

5. Implement the VS Code Copilot adapter in two passes:
First, discovery/listing across likely user-data roots.

6. Make the adapter probe both classes of storage:
The generic VS Code chat session roots described by the VS Code source.
The GitHub Copilot Chat extension storage that already contains transcript JSONL files on this machine.
7. Add story-focused CLI commands instead of jumping to commit hooks immediately:
A command to discover available provider sources.
A command to list sessions for a provider.
